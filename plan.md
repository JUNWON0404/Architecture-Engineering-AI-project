# 프로젝트 개발 계획 (plan.md)

이 파일은 프로젝트의 구조, 주요 폴더/파일 역할, 개발 시 참고해야 할 규칙 및 TODO, 개선점 등을 기록하는 문서입니다.

---

## 1. 프로젝트 구조 개요

- **client/**: 프론트엔드 리소스 (React, Vite 기반)
  - `src/pages/CoverLetterEditor.tsx`: 단일 마스터 자소서 관리 및 편집 허브
  - `src/pages/Dashboard.tsx`: 기업 탐색, 북마크 필터링, 주요 지표 통합 대시보드
- **server/**: 백엔드 코드 (Node.js 기반)
  - `routers.ts`: tRPC API 정의 (마스터 자소서 로직 포함)
- **drizzle/**: DB 마이그레이션 및 스키마 (PostgreSQL 기반)

---

## 2. 주요 개발 전략

### 2.1 단일 마스터 자소서 (Single Master Draft)
- 사용자의 모든 경험(브레인스토밍, STAR 에피소드 등)은 **단 하나의 마스터 문서**에 집대성됨.
- 여러 문서를 만드는 대신, 하나의 본체를 지속적으로 업데이트하고 이를 기반으로 기업별 맞춤형 버전을 파생시키는 구조.

### 2.2 직관적인 UI 및 정보 가시성
- **중앙 집중형 레이아웃**: 대시보드 및 상세 페이지 가로 너비를 제한(`max-w-6xl`)하여 정보 밀도를 높임.
- **통합 필터링**: 북마크 페이지를 별도로 두지 않고 대시보드 내 스탯 카드를 통해 즉시 필터링.
- **시원한 편집 환경**: 자소서 에디터의 폰트 크기와 입력 공간을 대폭 키워 스크롤 중심의 쾌적한 작성 경험 제공.

### 2.3 데이터 무결성 및 인프라
- **BigInt 타임스탬프**: Date.now() 밀리초 값을 안전하게 저장하기 위해 DB 전반에 `bigint` 적용.
- **PostgreSQL (Supabase)**: 클라우드 DB 기반의 안정적인 데이터 관리.
